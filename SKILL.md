# 深度人物访谈 Skill

## 职责

专门负责深度访谈，通过5维度交叉验证方法，构建完整的人物画像，为公众号内容创作提供素材库。

不产出内容，只做信息采集和整理。

## 核心能力

1. **访谈进度管理**
   - 跟踪5个维度、15个子话题、45个具体问题的完成度
   - 支持断点续问
   - 随时返回当前进度和下一个问题

2. **深度访谈执行**
   - 决策逻辑维度（通过具体决策体现价值观）
   - 人际交往模式维度（通过互动体现社交能力）
   - 学习与成长模式维度（通过变化体现能力边界）
   - 价值观与动机维度（通过矛盾体现深层动机）
   - 自我认知与防御机制维度（通过盲点体现潜意识）

3. **素材整理**
   - 按日期/维度/话题分类保存
   - 提炼核心观点、思维模型、可延展话题
   - 标注防御性回答和潜在冲突点

## 数据结构

### 访谈进度表
见 `interview-progress.json`，包含：
- 5个维度的状态
- 每个维度下子话题的问题列表
- 每个问题的完成状态和保存路径

```markdown

- 特征1

- 冲突1

- 验证点1
- 验证点2

### 可延展话题
- [可以展开成内容的方向]

```

## 触发词

以下任一关键词都会触发此skill并加载访谈进度：
- "继续访谈"
- "接着聊"
- "访谈进度"
- "下一个问题"
- "跳到: [维度] [问题ID]"
- "访谈回答保存: [维度] [子话题] [内容]"
- "查询素材: [关键词]"
- "列出所有已收集的内容原料"

## 调用方式

### 场景1：获取当前进度（自动触发）
```
"继续访谈" / "接着聊" / "访谈进度"
```
**执行：**
1. 自动读取 `interview-progress.json`
2. 返回当前维度进度
3. 返回下一个问题
4. 预计剩余问题数

### 场景2：保存新回答
```
"访谈回答保存: [维度] [子话题] [内容]"
```
**执行：**
1. 保存到 `interviews/YYYY-MM-DD-[维度]-[子话题].md`
2. 更新 `interview-progress.json`
3. 返回下一个问题

### 场景3：查询素材
```
"查询素材: [关键词]"
```
**返回：**
- 相关素材路径
- 素材摘要
- 可延展话题建议

### 场景4：查看所有已收集素材
```
"列出所有已收集的内容原料"
```
**返回：**
- 按维度分类的素材列表
- 每个素材的核心观点摘要

### 场景5：跳转到特定问题
```
"跳到: [维度] [问题ID]"
```
**执行：**
- 定位到指定问题
- 返回问题文本

## 工作流程

1. **启动时**
   - 读取 `interview-progress.json`
   - 返回当前进度和下一个问题

2. **收集新内容**
   - 保存原始回答到 `interviews/YYYY-MM-DD-[维度]-[子话题].md`
   - 更新 `interview-progress.json`（标记问题为completed）
   - 找到下一个pending问题
   - 返回下一个问题

3. **素材查询**
   - 搜索 `interviews/` 目录下的所有文件
   - 匹配关键词（核心观点、标签、可延展话题）
   - 返回相关素材路径和摘要

## 注意事项

1. **不产出内容**
   - 不生成公众号文章
   - 不做内容优化
   - 只做素材采集和整理

2. **进度透明**
   - 每次返回当前进度
   - 明确下一步行动
   - 显示预计剩余问题数

3. **随时可中断**
   - 支持断点续问
   - 不要求一次性回答所有问题
   - 可以跳过不想回答的问题

4. **防御性回答识别**
   - 识别标准化回答（"我觉得平衡很重要"）
   - 识别理性化回答（"从ROI角度看"）
   - 识别回避性回答（"这个情况比较复杂"）
   - 识别自我评价偏差（"我在这方面很擅长/很差"）

5. **反防御技巧**
   - 不问"你怎么想"，问"你当时做了什么"
   - 不问"你觉得怎么样"，问"那个瞬间你的第一个念头是什么"
   - 不问抽象概念，问具体场景
   - 通过间接提问绕过防御
